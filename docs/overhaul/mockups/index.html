KillrVideo

UI/UX Overhaul — Prototypes

Six design directions for the redesigned KillrVideo developer experience. Each prototype explores a different approach to blending a consumer video interface with always-available developer tooling — API traces, CQL/DataAPI switching, multi-language code samples, and live schema inspection.

Design Directions

01

Terminal Fusion

Dark IDE aesthetic with a persistent left sidebar dev panel. API traces, schema, and code samples are always visible alongside the video UI.

Best for: developers who want always-on tooling

View Prototype →

02

Clean Canvas

Minimal white design with inline expandable dev blocks. Dev context appears in-place only when requested, keeping the UI distraction-free.

Best for: casual exploration, lowest learning curve

View Prototype →

03

Split Lens

50/50 split screen with the consumer UI on the left and the full dev panel on the right. Both panes scroll independently.

Best for: deep study and side-by-side comparison

View Prototype →

04

Floating Inspector

A draggable floating dev panel overlaid on a refined consumer UI. Minimise, move, or resize it without leaving the page.

Best for: power users who want full layout control

View Prototype →

05

Narrative Scroll

Editorial layout with dev explainer bands woven between content sections. Each scroll step reveals a new layer of how the app works.

Best for: first-time learners, guided narrative

View Prototype →

06 — CANDIDATE

Learning Split

Split-pane layout with the consumer side faithfully matching the real KillrVideo app — purple hero, video cards, comments, related videos — paired with a dark dev panel on the right.

Best for: realistic implementation candidate

View Prototype →

How to Use These Prototypes

Each prototype is a fully self-contained HTML file — no build step required. Open any of them directly in a browser and explore the following interactions:

  • 1 Toggle dev mode — look for a "Dev Mode" toggle or button to show and hide the developer overlay or panel.
  • 2 Switch between CQL and Data API — use the API-mode tabs to see how the same operation looks in Cassandra Query Language vs. the Astra Data API.
  • 3 Try different language tabs — code samples are shown in multiple languages (Python, JavaScript, Java, curl). Click each tab to switch.
  • 4 Switch between Home and Watch views — each prototype renders at least two page contexts so you can see how the dev panel adapts across different routes.
