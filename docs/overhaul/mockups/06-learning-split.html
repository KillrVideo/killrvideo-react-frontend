Schema — latest_videos
yyyymmdd TEXT PK
added_date TIMESTAMP CK ↓

Design note: Bucketed by date (yyyymmdd) so each partition stays bounded. Without bucketing, a single "latest_videos" partition would grow unbounded — a classic Cassandra anti-pattern.
